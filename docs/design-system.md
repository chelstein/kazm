# KAZM Design System

## Purpose

The KAZM Design System exists to ensure every page, component, feature, and user experience feels like KAZM.

This system prioritizes:

* Radio first
* Community first
* Utility first
* Accessibility first

The goal is consistency, trust, and usability.

Not novelty.

Not trends.

Not design awards.

---

# Design Principles

## Radio First

The radio signal is the product.

Every design decision should strengthen the connection between listeners and the station.

Listen Live should always be easy to find.

Now Playing should always be visible.

The station is the destination.

---

## Local Before Corporate

Visitors should immediately understand:

"This is Sedona's radio station."

The website should feel:

* Regional
* Personal
* Community-focused
* Authentic

Never:

* Corporate
* Generic
* Anonymous

---

## Utility Before Spectacle

Choose:

* Readability
* Clarity
* Speed
* Accessibility

Over:

* Visual effects
* Animation
* Trends
* Complexity

Useful beats beautiful.

---

## Timeless Before Trendy

The website should still feel appropriate five years from now.

Avoid:

* Design fads
* Startup aesthetics
* Excessive animation
* Social media inspired layouts

Favor longevity.

---

# Brand Personality

The website should feel:

* Warm
* Relaxed
* Friendly
* Welcoming
* Authentic
* Music-loving
* Arizona proud
* Community-focused

The website should never feel:

* Sterile
* Corporate
* Academic
* Technical
* Futuristic
* Aggressive

---

# Color System

## Primary Colors

### KAZM Sky Blue

Inspired by Arizona skies.

Used for:

* Links
* Interactive elements
* Buttons
* Navigation highlights

### KAZM Red Rock

Inspired by Sedona sandstone.

Used for:

* Brand accents
* Highlights
* Calls to action
* Weather alerts

### Desert Earth

Used for:

* Background accents
* Supporting graphics

### Juniper Green

Used sparingly for:

* Nature content
* Community content
* Supporting accents

---

## Color Rules

Colors support content.

Colors should never dominate content.

Avoid:

* Neon palettes
* Startup gradients
* Purple AI themes
* Cyberpunk aesthetics

The palette should feel naturally Arizona.

---

# Typography

## Typography Philosophy

Typography should prioritize readability.

Not personality.

Not trendiness.

Not novelty.

Readers should never struggle to consume content.

---

## Headlines

Headlines should be:

* Large
* Clear
* High contrast

Headlines should communicate importance.

---

## Body Text

Body text should be:

* Comfortable
* Readable
* Accessible

Avoid:

* Small fonts
* Low contrast
* Dense paragraphs

---

# Layout System

## Maximum Width

Content should remain comfortable to read.

Avoid excessively wide layouts.

Text should never stretch across the entire screen.

---

## Spacing

Favor generous whitespace.

Whitespace should improve readability.

Whitespace should never feel empty.

---

## Grid Philosophy

Use simple predictable layouts.

Avoid:

* Experimental grids
* Broken layouts
* Trend-driven asymmetry

Users should instantly understand page structure.

---

# Buttons

## Primary Buttons

Used for:

* Listen Live
* Stream actions
* Emergency actions
* Contact actions

Primary buttons should be visually obvious.

---

## Secondary Buttons

Used for:

* Supporting navigation
* Additional actions

Secondary buttons should never compete with Listen Live.

---

# Cards

Cards should be used for:

* News stories
* Weather information
* Sports coverage
* Events
* Advertisers

Cards should:

* Be readable
* Be consistent
* Be mobile friendly

Avoid decorative cards.

---

# Navigation

Navigation should be obvious.

Primary navigation:

* Listen Live
* News
* Weather
* Sports
* Schedule
* Events
* Advertise
* About
* Contact

Avoid:

* Hidden navigation
* Clever navigation
* Experimental navigation

If navigation requires explanation, redesign it.

---

# Homepage Components

Required Homepage Components:

1. KAZM Logo
2. Listen Live
3. Now Playing
4. Weather
5. Breaking News
6. Local News
7. Sports
8. Schedule Preview
9. Events
10. Advertise CTA

Nothing should outrank these components.

---

# Motion System

## Motion Philosophy

Motion should support content.

Motion should never become content.

---

## Allowed Motion

* Smooth fades
* Gentle transitions
* Hover states
* Subtle reveals
* Now Playing updates
* Weather updates

---

## Forbidden Motion

* Scroll-jacking
* Full-screen intros
* Cursor effects
* Magnetic buttons
* Excessive parallax
* Frame-by-frame scrolling
* Animation that delays information

Visitors came for information.

Not a movie trailer.

---

# Photography

## Preferred Photography

Use:

* Real station photography
* Real staff photography
* Real local photography
* Community events
* Sedona landscapes
* Local businesses

---

## Avoid

* Generic stock photography
* AI-generated people
* Fake imagery
* Corporate office imagery

Authenticity always wins.

---

# Accessibility Standards

Every component must support:

* WCAG compliance
* Keyboard navigation
* Screen readers
* Proper contrast
* Responsive layouts

Accessibility is mandatory.

---

# Performance Standards

Prioritize:

* Fast loading
* Optimized images
* Minimal JavaScript
* Mobile performance

Performance is a feature.

---

# Anti-Slop Rules

Never generate:

* SaaS components
* Startup hero sections
* Feature grids
* Corporate marketing layouts
* Dashboard interfaces
* Empty visual sections
* Generic AI website patterns

Never describe KAZM as:

* A platform
* A solution
* An ecosystem
* A digital experience platform

KAZM is a radio station.

Design like it.

---

# Final Design Test

Before approving any component ask:

Does this help listeners?

Does this strengthen trust?

Does this feel like Sedona?

Does this feel like KAZM?

Would a longtime listener recognize this as their station?

If any answer is no:

Redesign it.

Simplify it.

Or remove it.

Do not ship it.
